## 확정된 설계 결정 (뒤집으려면 근거 필요)

1. **키워드 격자 수집** (채널 방식 아님 — 넓은 도메인엔 키워드가 맞음, 실측 검증됨)
2. **무손실 raw**: 설명란 0자도 저장. "정보 없는 데이터도 투표권(다수결)은 있다"
3. **이원 트랙**: A(설명란 有→슬롯 추출) / B(숏츠, 제목+태그로 카페명 특정)
4. **info_richness 3단**: low는 검색 오염 방지 위해 임베딩 제외, mention 신호로만 병합
5. **video_id 등 식별자는 LLM 우회** — 코드가 운반 (환각 방지)
6. **런타임 3단계 체인** (에이전트 아님): [1]질문 재작성 → [2]검색+필터 → [3]셋리스트 생성
   - [1]은 필수로 검증됨: "클럽이 많은 곳" → 승마클럽 사건 (표면 토큰 함정)
7. **mention_count(영상 수) + channel_count(채널 수) 이원화** — 채널 교차가 진짜 다수결
8. **점수의 자리**: 인기(차트인)는 검색이 아니라 동점 정렬+카피에만 (인기≠적합성)
9. **카카오 로컬 = Pass 2 대체**: place_id가 병합 키 + 실존검증 + 좌표 (경로1: 주소 393건)
10. **공공 카페 명부(~2,000개)** = 검증용 뼈대, 유튜브 = 추천 신호 (조인 방향: 유튜브→명부)

## 그동안 밟은 함정들 (재발 방지)

- gpt-5-mini는 추론 모델: max_completion_tokens 부족 시 **빈 응답** → 4000 + minimal
- except가 에러를 삼키면 전멸이 0개처럼 보임 → 경고 출력 필수
- 관통 표본은 코퍼스 머리가 아니라 **random.sample** (머리는 수집 이력순 편향)
- 유튜브 태그의 지역명은 스팸, 고유 장소명은 채택
- 엑셀용 CSV는 utf-8-sig (아니면 한글 깨짐)
- VS Code 실행 디렉토리 ≠ 스크립트 위치 → 경로는 ROOT 상수로 고정할 것
